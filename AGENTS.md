# 前端项目编码规范

## 1. 样式规范

- **BEM**：核心是「块__元素 -- 修饰符」的命名结构，Less 中嵌套编写时禁止多层元素嵌套，块 / 元素 / 修饰符命名需语义化；
- **通用样式**：需使用全局变量文件（如:ThemeVariable.css、SystemVariable.css）中的变量，避免重复定义，保障性能与可维护性。

## 2. 组件封装规范

- **核心规范**：组件命名默认大驼峰（特殊场景按要求调整），Props / 事件用 TS 强类型定义，样式用 Less + BEM 规范。
- **代码简化**：无需重复引入 defineProps/withDefaults，直接使用 Vue 内置的语法糖。
- **文档要求**：注释需包含详细使用示例，覆盖基础用法和常见定制场景，提升组件易用性。

## 3. 业务逻辑规范

- **事件处理**：统一使用小驼峰命名法，命名前缀为 handle，示例：handleSubmitForm、handleFullScreenClick。
- **业务注释**：代码需补充清晰的业务注释，简要说明功能用途与核心实现逻辑，提升可读性与可维护性。

## 4. json数据格式

- **字段**：以小驼峰格式为准，小驼峰命名的核心是首字母小写，后续单词首字母大写，无分隔符；
- **组件参数格式**：所有组件参数在 codeView 字段中均使用小驼峰命名法，包括但不限于：`largeCols`（替代 `large-cols`）

## 5. 状态管理

- **设计**：单一数据源。
- **Mutation**：大写加下划线，如 `SET_USER_INFO`。
- **Action**：小驼峰，如 `fetchUserInfo`。

## 6. 性能优化

- **渲染**：合理使用 v-if 和 v-show 指令，避免渲染不必要的 DOM 元素。
- **计算**：复杂逻辑用 computed 计算属性，避免在模板中重复计算。
- **清理**：组件销毁时清理监听器，避免内存泄漏。

## 7. 其他规则

**开发流程规范：**

- 项目已经在浏览器中启动，不需要每次完成任务后都重新启动项目进行确认
- 无论是编写单页面、封装组件、正常任务编写还是安装依赖等操作，都不需要重新启动项目
- 完成业务需求后，不需要每次打开预览浏览器，因为已经在浏览器中能够实时看到页面
- 只有在必要的情况下（如配置文件变更需要重启服务）才会启动项目

**样式编码规范：**

- 样式使用全局变量，不写 fallback 后缀，如 `background: var(--background)`、`border: var(--size-1) solid var(--border)`
- 禁止在 CSS 变量中使用带单位或颜色值的 fallback，如 `var(--fs-24, 24px)`、`var(--background-secondary, #f5f7fa)` 等形式
- 所有样式变量应在全局变量文件中定义，组件中直接使用，确保样式一致性和可维护性
