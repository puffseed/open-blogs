# 四、详细功能需求（与Axure原型对应）

## 4.1 基础配置模块

### 4.1.1 字典管理

【功能描述】：用于管理系统中所有字典数据（如：订单状态、用户类型、商品分类等），支持字典的新增、编辑、删除、查询，确保系统数据标准化。

【原型对应】：Axure原型中「基础配置-字典管理」页面，包含字典列表、新增字典弹窗、编辑字典弹窗。

【交互要求】（Axure需实现）：

- 点击「新增字典」按钮，弹出新增弹窗，填写字典名称、字典编码、备注后，点击确认提交，新增成功后列表刷新，提示“新增成功”；若填写不完整（如字典名称为空），提示对应错误信息。

- 点击列表中「编辑」按钮，弹出编辑弹窗，自动填充当前字典信息，修改后提交，列表刷新，提示“编辑成功”；字典编码不可修改。

- 点击列表中「删除」按钮，弹出确认弹窗（提示“确定删除该字典吗？删除后不可恢复”），点击确认删除，列表刷新，提示“删除成功”；点击取消，关闭弹窗。

- 列表支持按「字典名称、字典编码」模糊查询，输入关键词后点击查询按钮，列表显示匹配结果；点击重置按钮，清空查询条件，列表恢复默认显示。

- 列表支持分页，每页显示10条数据，可切换页码、跳转至指定页码。

【字段要求】：

|字段名称|字段类型|是否必填|备注|
|---|---|---|---|
|字典名称|文本|是|长度不超过50字符|
|字典编码|文本|是|唯一，长度不超过30字符，仅支持字母、数字、下划线|
|备注|文本域|否|长度不超过200字符|
### 4.1.2 区域管理

【功能描述】：管理系统中所有区域数据（如：省、市、区/县），支持区域新增、编辑、删除、树形展示，用于关联业务中的区域筛选、定位。

【原型对应】：Axure原型中「基础配置-区域管理」页面，包含区域树形列表、新增区域弹窗、编辑区域弹窗。

【交互要求】（Axure需实现）：

- 页面默认显示区域树形结构，支持展开/折叠父级区域，点击区域名称，右侧显示该区域详情（包含区域编码、父级区域、备注）。

- 点击「新增区域」按钮，弹出新增弹窗，选择父级区域（可选择顶级区域，即无父级），填写区域名称、区域编码、备注，提交后树形列表刷新，提示“新增成功”；必填项为空时提示错误。

- 点击树形列表中「编辑」按钮，弹出编辑弹窗，自动填充当前区域信息，可修改区域名称、备注（父级区域、区域编码不可修改），提交后刷新，提示“编辑成功”。

- 点击「删除」按钮，若该区域无下级区域，弹出确认弹窗，确认后删除，提示“删除成功”；若有下级区域，提示“该区域存在下级区域，无法删除”。

- 支持按区域名称模糊查询，输入关键词后，树形列表显示匹配的区域及父级区域。

### 4.1.3 参数配置

【功能描述】：配置系统全局参数（如：默认分页条数、文件上传大小限制、接口超时时间等），支持参数编辑、保存，参数生效后作用于整个系统。

【原型对应】：Axure原型中「基础配置-参数配置」页面，包含参数列表、编辑参数弹窗。

【交互要求】（Axure需实现）：

- 列表显示参数名称、参数键、参数值、备注、更新时间，支持按参数名称、参数键模糊查询。

- 点击「编辑」按钮，弹出编辑弹窗，自动填充当前参数信息，可修改参数值、备注，提交后列表刷新，提示“参数修改成功”；参数键不可修改。

- 参数值需根据参数类型进行校验（如：数字类型参数，输入非数字时提示“请输入有效的数字”）。

### 4.1.4 公告管理

【功能描述】：管理后台系统公告（如：系统更新通知、业务调整通知等），支持公告新增、编辑、删除、发布/下架，公告将在后台首页展示。

【原型对应】：Axure原型中「基础配置-公告管理」页面，包含公告列表、新增公告弹窗、编辑公告弹窗。

【交互要求】（Axure需实现）：

- 点击「新增公告」，弹出弹窗，填写公告标题、公告内容、发布人、发布时间（默认当前时间，可修改），选择发布状态（未发布/已发布），提交后列表刷新，提示“新增成功”。

- 点击「编辑」，弹出弹窗，可修改所有字段（发布时间可修改），提交后刷新，提示“编辑成功”。

- 点击「删除」，弹出确认弹窗，确认后删除，提示“删除成功”；已发布的公告可正常删除。

- 列表支持按「公告标题、发布人、发布状态」筛选，支持按发布时间排序（默认倒序）。

- 点击公告标题，可查看公告详情（弹窗展示完整公告内容）。

## 4.2 用户权限模块

### 4.2.1 用户管理

【功能描述】：管理后台所有操作用户，支持用户新增、编辑、删除、启用/禁用、密码重置，分配用户角色，控制用户操作权限。

【原型对应】：Axure原型中「权限管理-用户管理」页面，包含用户列表、新增用户弹窗、编辑用户弹窗、分配角色弹窗、密码重置弹窗。

【交互要求】（Axure需实现）：

- 点击「新增用户」，弹出弹窗，填写用户名、账号、密码（默认生成随机密码，可修改）、手机号、邮箱、所属部门，选择用户状态（启用/禁用），分配角色（可多选），提交后列表刷新，提示“新增成功”；账号唯一，必填项为空时提示错误。

- 点击「编辑」，弹出弹窗，可修改用户名、手机号、邮箱、所属部门、用户状态、角色，提交后刷新，提示“编辑成功”；账号不可修改。

- 点击「密码重置」，弹出弹窗，提示“重置后密码为默认密码（123456），请提醒用户及时修改”，确认后重置密码，提示“密码重置成功”。

- 点击「启用/禁用」，切换用户状态，提示“用户已启用”/“用户已禁用”；禁用后的用户无法登录后台。

- 点击「删除」，弹出确认弹窗，确认后删除，提示“删除成功”；超级管理员不可删除。

- 列表支持按「用户名、账号、所属部门、用户状态」筛选，支持分页、模糊查询。

### 4.2.2 角色管理

【功能描述】：管理后台角色（如：超级管理员、运营管理员、财务管理员等），支持角色新增、编辑、删除，为角色分配权限（关联功能模块），实现权限批量管理。

【原型对应】：Axure原型中「权限管理-角色管理」页面，包含角色列表、新增角色弹窗、编辑角色弹窗、权限分配弹窗。

【交互要求】（Axure需实现）：

- 点击「新增角色」，弹出弹窗，填写角色名称、角色描述，提交后列表刷新，提示“新增成功”；角色名称唯一，必填项不能为空。

- 点击「编辑」，弹出弹窗，可修改角色名称、角色描述，提交后刷新，提示“编辑成功”。

- 点击「权限分配」，弹出弹窗，显示所有功能模块的权限（勾选框形式），可勾选对应模块的权限（如：查看、新增、编辑、删除），提交后提示“权限分配成功”；超级管理员角色权限不可修改。

- 点击「删除」，弹出确认弹窗，确认后删除，提示“删除成功”；超级管理员角色不可删除；若该角色已关联用户，提示“该角色已分配给用户，无法删除”。

- 列表支持按角色名称模糊查询，支持分页。

### 4.2.3 权限管理

【功能描述】：查看系统所有权限节点，支持权限节点的新增、编辑、删除（仅超级管理员可操作），用于角色权限分配的基础数据支撑。

【原型对应】：Axure原型中「权限管理-权限节点」页面，包含权限节点树形列表、新增节点弹窗、编辑节点弹窗。

【交互要求】（Axure需实现）：

- 页面默认显示权限节点树形结构（按模块分类，如：基础配置、用户权限等），支持展开/折叠。

- 点击「新增节点」，弹出弹窗，选择父级节点，填写节点名称、节点标识、节点路径（对应原型页面路径）、备注，提交后树形列表刷新，提示“新增成功”。

- 点击「编辑」，弹出弹窗，可修改节点名称、节点路径、备注，提交后刷新，提示“编辑成功”；节点标识不可修改。

- 点击「删除」，弹出确认弹窗，确认后删除，提示“删除成功”；若该节点有子节点，提示“该节点存在子节点，无法删除”。

## 4.3 业务管理模块（可根据具体业务修改）

说明：本模块为B端后台核心业务模块，需根据实际业务场景调整功能，以下以「订单管理」为例，提供标准化模板，其他业务模块（如客户管理、商品管理等）可参考此格式编写。

### 4.3.1 订单管理

【功能描述】：管理所有业务订单，支持订单查询、详情查看、订单编辑、订单状态修改、订单导出，处理订单相关异常（如：取消订单、退款订单）。

【原型对应】：Axure原型中「业务管理-订单管理」页面，包含订单列表、订单详情页、编辑订单弹窗、状态修改弹窗。

【交互要求】（Axure需实现）：

- 列表显示订单编号、下单用户、订单金额、订单状态、下单时间、支付时间、操作人，支持多条件筛选（订单状态、下单时间范围、订单编号、下单用户）。

- 支持订单导出，点击「导出」按钮，导出当前筛选条件下的所有订单数据（Excel格式），提示“导出成功，请注意查收”。

- 点击订单编号，跳转至订单详情页，详情页显示订单基本信息、商品信息、支付信息、物流信息（如有）、操作日志。

- 点击「编辑」，弹出编辑弹窗，可修改订单备注、收货信息（如：收货地址、手机号），提交后列表刷新，提示“编辑成功”；订单金额、下单时间不可修改。

- 点击「修改状态」，弹出弹窗，选择目标状态（如：待支付→已支付、已发货→已完成、待支付→已取消），填写状态修改原因，提交后列表刷新，提示“状态修改成功”；状态流转需符合业务规则（如：已完成订单不可修改为待支付）。

- 列表支持分页、按下单时间排序（默认倒序），支持模糊查询（订单编号、下单用户）。

【字段要求】：

|字段名称|字段类型|是否必填|备注|
|---|---|---|---|
|订单编号|文本|是|唯一，系统自动生成（格式：YYYYMMDD+随机6位数字）|
|下单用户|文本|是|关联用户表，显示用户名/用户ID|
|订单金额|数字|是|保留2位小数，单位：元|
|订单状态|下拉选择|是|选项：待支付、已支付、已发货、已完成、已取消、退款中、已退款|
|下单时间|时间|是|系统自动生成，格式：YYYY-MM-DD HH:MM:SS|
|备注|文本域|否|长度不超过200字符|
## 4.4 数据统计模块

### 4.4.1 核心数据仪表盘

【功能描述】：展示系统核心数据指标，以图表（折线图、柱状图、饼图）形式呈现，支持数据筛选（时间范围），帮助管理员快速掌握业务概况。

【原型对应】：Axure原型中「数据中心-核心仪表盘」页面，包含数据概览卡片、各类图表。

【交互要求】（Axure需实现）：

- 页面顶部显示核心数据卡片（如：总用户数、今日新增用户、总订单数、今日订单数、订单总金额、今日营收），数据实时刷新（原型中可模拟静态数据，标注“数据实时更新”）。

- 支持时间范围筛选（今日、昨日、近7天、近30天、自定义时间），选择时间范围后，所有图表、数据卡片同步更新。

- 包含核心图表：
        
1.  用户增长趋势图（折线图）：展示指定时间范围内的用户新增趋势；
        
2.  订单量趋势图（柱状图）：展示指定时间范围内的每日订单量；
        
3.  订单状态分布（饼图）：展示当前所有订单的状态占比；
       
4.  营收趋势图（折线图）：展示指定时间范围内的每日营收。


- 鼠标悬浮在图表上，显示具体数据（如：悬浮在折线图某点，显示“日期：202X-MM-DD，新增用户：XX人”）。

### 4.4.2 业务数据报表

【功能描述】：生成各类业务明细报表（如：用户报表、订单报表、营收报表），支持报表筛选、导出，用于数据统计、审计、分析。

【原型对应】：Axure原型中「数据中心-业务报表」页面，包含报表类型切换、筛选条件、报表列表、图表。

【交互要求】（Axure需实现）：

- 支持报表类型切换（用户报表、订单报表、营收报表），切换后筛选条件、报表内容、图表同步切换。

- 每个报表支持多条件筛选（如：用户报表可按用户类型、注册时间筛选；订单报表可按订单状态、下单时间、用户类型筛选）。

- 报表列表显示对应明细数据，支持分页、排序；点击「导出报表」，导出当前筛选条件下的报表数据（Excel格式），提示“导出成功”。

- 每个报表对应一个辅助图表（如：用户报表对应用户类型分布饼图，营收报表对应营收趋势柱状图），图表与报表数据同步，支持时间范围筛选。

## 4.5 系统设置模块

### 4.5.1 系统日志

【功能描述】：记录系统所有操作日志（如：用户登录、新增用户、修改订单、修改参数等），支持日志查询、筛选，用于系统审计、问题排查。

【原型对应】：Axure原型中「系统设置-系统日志」页面，包含日志列表、筛选条件。

【交互要求】（Axure需实现）：

- 日志列表显示操作人、操作内容、操作时间、操作IP、操作结果（成功/失败）。

- 支持多条件筛选：操作人、操作时间范围、操作结果、操作内容（模糊查询）。

- 支持分页、按操作时间倒序排序（默认）；日志不可编辑、不可删除。

### 4.5.2 备份与恢复

【功能描述】：实现系统数据备份（手动备份、自动备份）、数据恢复，防止数据丢失，保障系统稳定运行。

【原型对应】：Axure原型中「系统设置-备份与恢复」页面，包含备份列表、手动备份按钮、恢复按钮。

【交互要求】（Axure需实现）：

- 备份列表显示备份时间、备份类型（手动/自动）、备份大小、操作人。

- 点击「手动备份」，弹出确认弹窗，提示“手动备份将耗时1-5分钟，请耐心等待”，确认后开始备份，备份完成后列表刷新，提示“备份成功”。

- 点击「恢复」，弹出确认弹窗，提示“恢复数据将覆盖当前系统所有数据，且不可恢复，请确认是否继续”，确认后开始恢复，恢复完成后提示“恢复成功，请重启系统生效”。

- 支持自动备份设置（点击「设置」按钮，弹出弹窗，设置自动备份时间、备份频率（每日/每周/每月），提交后生效）。

### 4.5.3 接口配置

【功能描述】：配置系统与外部系统的接口（如：对接用户端APP、数据中台、第三方支付接口），支持接口新增、编辑、删除、测试，确保接口正常通信。

【原型对应】：Axure原型中「系统设置-接口配置」页面，包含接口列表、新增接口弹窗、编辑接口弹窗、接口测试弹窗。

【交互要求】（Axure需实现）：

- 点击「新增接口」，弹出弹窗，填写接口名称、接口地址、请求方式（GET/POST）、请求参数、响应参数、备注，提交后列表刷新，提示“新增成功”。

- 点击「编辑」，弹出弹窗，可修改所有字段，提交后刷新，提示“编辑成功”。

- 点击「测试」，弹出测试弹窗，自动填充接口信息，点击「发起测试」，显示测试结果（成功/失败）、响应数据；测试失败时，显示错误信息。

- 点击「删除」，弹出确认弹窗，确认后删除，提示“删除成功”；已关联业务的接口，提示“该接口已关联业务，无法删除”。

- 列表支持按接口名称、接口地址模糊查询，支持按接口状态（启用/禁用）筛选。

### 4.5.4 个性化设置

【功能描述】：配置后台系统个性化信息（如：系统标题、logo、主题颜色、首页布局），支持保存生效，提升用户操作体验。

【原型对应】：Axure原型中「系统设置-个性化设置」页面，包含各项设置项、保存按钮。

【交互要求】（Axure需实现）：

- 支持系统标题修改（输入文本，实时预览）、logo上传（点击上传按钮，选择图片，预览效果）。

- 支持主题颜色切换（提供多种预设主题，点击切换，实时生效）、首页布局切换（如：仪表盘在上/在下、菜单在左/在顶）。

- 所有设置修改后，点击「保存设置」，提示“保存成功”，刷新页面后生效；点击「重置默认」，恢复系统默认设置，提示“已重置为默认设置”。

# 六、异常场景处理（Axure需实现）

- 登录异常：账号密码错误、账号禁用、账号未分配角色，提示对应错误信息（如：“账号或密码错误，请重新输入”）。

- 权限异常：用户访问无权限的页面/操作无权限的功能，提示“您无此操作权限，请联系管理员”，并跳转至首页。

- 数据异常：数据加载失败、数据为空（如：列表无数据时，显示“暂无数据”）、数据重复（如：新增时输入重复账号，提示“该账号已存在”）。

- 操作异常：删除有关联数据的记录（如：删除关联用户的角色）、修改不可修改的字段（如：订单金额），提示对应错误信息；操作超时，提示“操作超时，请重试”。
